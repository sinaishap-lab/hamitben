# 🌾 המתבן – אפליקציית השאלת כלי עבודה חקלאיים
## מסמך תכנון מלא לביצוע בקלוד קוד
### גרסה 2.0 – מעודכן לאחר סקר דרישות מלא

---

## 1. סקירת הפרוייקט

**שם:** המתבן  
**מטרה:** פלטפורמה לניהול גמחי כלי עבודה חקלאיים – השאלה, תשלום ופיקדון בכרטיס אשראי, ניהול גמחים וכלים.  
**שפה:** עברית בלבד (RTL מלא)  
**פלטפורמה ראשית:** מובייל (mobile-first, responsive)  
**גמחים בהשקה:** מזרח הגוש | תלם | דרום הר חברון

---

## 2. טכנולוגיות (Stack)

| שכבה | טכנולוגיה | הערה |
|------|-----------|------|
| Frontend | **Next.js 14** (App Router) + TypeScript | Full-stack, SSR, routing מובנה |
| Styling | **Tailwind CSS** + shadcn/ui | RTL מלא, mobile-first |
| Database | **PostgreSQL** + **Prisma ORM** | type-safe, relational |
| Auth | **NextAuth.js** + bcrypt | credentials, session |
| File Storage | **Cloudinary** | תמונות כלים, אופטימיזציה |
| WhatsApp | **Twilio** WhatsApp API | הודעות מצב |
| Push | **web-push** / Vercel Notifications | התראות דחיפה בדפדפן |
| Email | **Resend** | אימיילים transactional |
| SMS | **Twilio SMS** | SMS כגיבוי |
| תשלומים | **Cardcom** | סליקה ישראלית + נעילת פיקדון |
| Maps | **Google Maps API** | מיקום גמחים |
| Hosting | **Vercel** + **Supabase** (DB) | פשטות + סקלה |

---

## 3. תפקידים ורמות הרשאה

| תפקיד | יכולות |
|--------|--------|
| **אורח** | צפייה בקטלוג, חיפוש, דף גמח – אינו יכול להשאיל |
| **משתמש רגיל** (APPROVED) | כל הנ"ל + בקשת השאלה, ביטול בקשה, דירוג, מחיקת חשבון |
| **מנהל גמח** | כל הנ"ל + ניהול כלים בגמח שלו, אישור/דחיית בקשות, שחרור/גבייה פיקדון |
| **מנהל ראשי** | כל הנ"ל + ניהול כל הגמחים, אישור משתמשים, הוספת כלים לכל גמח |

> ⚠️ **השאלה מותרת רק למשתמשים עם status=APPROVED.**  
> ⚠️ **משתמש שהחזיר כלי באיחור → חסום מהשאלה עתידית לחלוטין** (ניתן לבטל חסימה ע"י מנהל ראשי).

## 4. זרימת מחזור חיי השאלה

```
משתמש בוחר כלי + תאריכים
        │
        ▼
 בדיקת זמינות בלוח שנה
        │
        ▼
 מילוי טופס: תאריך התחלה + תאריך סיום + מטרה (אופציונלי)
 (שם + טלפון + אימייל נלקחים אוטומטית מהפרופיל)
        │
        ▼
 נעילת פיקדון בכרטיס אשראי (Cardcom pre-auth)
        │
  ┌─────┴─────┐
  │           │
אוטומטי     ידני
(לפי הגדרת  (מנהל גמח
  הכלי)       מאשר)
  │           │
  └─────┬─────┘
        │ APPROVED
        ▼
 WhatsApp + Push + SMS + Email → משתמש
        │
        ▼
  משתמש אוסף כלי פיזית מהגמח
        │
        ▼
  מנהל גמח מסמן "כלי נאסף" (ACTIVE)
        │
        ▼
  תאריך סיום מתקרב → תזכורת אוטומטית
        │
        ▼
  משתמש מחזיר כלי פיזית
        │
  ┌─────┴──────────────────┐
  │                        │
תקין                   לא תקין / באיחור
  │                        │
מנהל גמח מסמן           מנהל גמח מסמן
"הוחזר תקין"            "נזק / איחור"
  │                        │
שחרור פיקדון            גביית פיקדון (Cardcom)
  │                        │
חיוב לפי ימים           + חסימת משתמש
בפועל (Cardcom)          מהשאלה עתידית
```

## 5. סכמת בסיס נתונים (Prisma)

```prisma
generator client {

datasource db {
  provider = "postgresql"

// ───────────────────────────────
// USERS
// ───────────────────────────────
model User {
  id           String     @id @default(cuid())
  name         String
  phone        String     @unique
  email        String     @unique
  passwordHash String
  role         UserRole   @default(REGULAR)
  status       UserStatus @default(PENDING)
  isBanned     Boolean    @default(false)   // חסום עקב איחור
  banReason    String?
  deletedAt    DateTime?                    // soft delete
  createdAt    DateTime   @default(now())
  updatedAt    DateTime   @updatedAt

  managedGemach Gemach?   @relation("GemachManager")
  loans         Loan[]
  reviews       Review[]
  pushTokens    PushToken[]
}

enum UserRole {
  REGULAR
  GEMACH_MANAGER
  ADMIN
}

enum UserStatus {
  PENDING   // ממתין לאישור מנהל ראשי
  APPROVED
  REJECTED
  SUSPENDED
}

// ───────────────────────────────
// GEMACHS
// ───────────────────────────────
model Gemach {
  id          String   @id @default(cuid())
  name        String   @unique
  description String?
  address     String
  lat         Float?
  lng         Float?
  phone       String?
  imageUrl    String?
  isActive    Boolean  @default(true)
  createdAt   DateTime @default(now())

  managerId String @unique
  manager   User   @relation("GemachManager", fields: [managerId], references: [id])
  tools     Tool[]
  reviews   Review[]
}

// ───────────────────────────────
// TOOLS
// ───────────────────────────────
model Tool {
  id            String       @id @default(cuid())
  name          String
  description   String?
  category      ToolCategory
  images        String[]     // Cloudinary URLs
  status        ToolStatus   @default(AVAILABLE)
  autoApprove   Boolean      @default(false)  // אישור אוטומטי לכלי זה
  depositAmount Float        @default(0)      // סכום פיקדון
  dailyRate     Float        @default(0)      // תעריף יומי
  maxDays       Int          @default(7)
  isActive      Boolean      @default(true)
  createdAt     DateTime     @default(now())
  updatedAt     DateTime     @updatedAt

  gemachId String
  gemach   Gemach @relation(fields: [gemachId], references: [id])
  loans    Loan[]
  reviews  Review[]
}

enum ToolCategory {
  IRRIGATION    // השקיה
  HARVESTING    // קציר ואסיף
  SOIL_WORK     // עיבוד קרקע
  SPRAYING      // ריסוס
  PLANTING      // שתילה
  STORAGE       // אחסון
  VEHICLES      // רכבים
  HAND_TOOLS    // כלי יד
  OTHER
}

enum ToolStatus {
  AVAILABLE
  LOANED
  MAINTENANCE
  INACTIVE
}

// ───────────────────────────────
// LOANS
// ───────────────────────────────
model Loan {
  id              String        @id @default(cuid())
  startDate       DateTime
  endDate         DateTime
  actualReturnDate DateTime?    // תאריך החזרה בפועל
  purpose         String?
  status          LoanStatus    @default(PENDING)
  depositStatus   DepositStatus @default(NONE)
  depositChargeId String?       // Cardcom pre-auth token
  paymentChargeId String?       // Cardcom final charge token
  depositAmount   Float         @default(0)
  dailyRate       Float         @default(0)
  totalCharged    Float?        // סכום סופי שחויב
  notes           String?
  adminNotes      String?       // הערות מנהל (סיבת דחייה וכו')
  createdAt       DateTime      @default(now())
  updatedAt       DateTime      @updatedAt

  toolId String
  tool   Tool   @relation(fields: [toolId], references: [id])
  userId String
  user   User   @relation(fields: [userId], references: [id])
}

enum LoanStatus {
  PENDING    // ממתין לאישור
  APPROVED   // אושר, טרם נאסף
  REJECTED   // נדחה
  ACTIVE     // נאסף, אצל השואל
  RETURNED   // הוחזר תקין
  OVERDUE    // הוחזר באיחור / נזק
  CANCELLED  // בוטל ע"י משתמש או מנהל
}

enum DepositStatus {
  NONE      // טרם נעשה פעולה
  HELD      // ננעל (pre-auth)
  RELEASED  // שוחרר בהצלחה
  CHARGED   // חויב (נזק/איחור)
}

// ───────────────────────────────
// REVIEWS
// ───────────────────────────────
model Review {
  id        String     @id @default(cuid())
  rating    Int        // 1–5
  comment   String?
  type      ReviewType
  createdAt DateTime   @default(now())

  userId   String
  user     User    @relation(fields: [userId], references: [id])
  toolId   String?
  tool     Tool?   @relation(fields: [toolId], references: [id])
  gemachId String?
  gemach   Gemach? @relation(fields: [gemachId], references: [id])
}

enum ReviewType {
  TOOL
  GEMACH
}

// ───────────────────────────────
// PUSH TOKENS (התראות דחיפה)
// ───────────────────────────────
model PushToken {
  id        String   @id @default(cuid())
  token     String   @unique
  platform  String   // web / android / ios
  createdAt DateTime @default(now())

  userId String
  user   User   @relation(fields: [userId], references: [id])
}

// ───────────────────────────────
// DONATIONS
// ───────────────────────────────
model Donation {
  id        String   @id @default(cuid())
  amount    Float
  status    String   @default("PENDING")
  chargeId  String?
  name      String?
  email     String?
  createdAt DateTime @default(now())
}
```

### 7.3 תשלומים (Cardcom)
```typescript

//   Cardcom pre-authorization על depositAmount
//   שמירת depositChargeId ב-Loan

// שלב 2 – בעת החזרה תקינה (מנהל גמח מסמן "הוחזר תקין"):
//   חיוב: dailyRate × actualDays → Cardcom charge
//   שחרור פיקדון: Cardcom void pre-auth
//   DepositStatus → RELEASED

// שלב 3 – בעת נזק/איחור (מנהל גמח מסמן "בעיה"):
//   גביית פיקדון: Cardcom capture pre-auth
//   DepositStatus → CHARGED
//   User.isBanned = true
```

### 7.4 מערכת התראות (notifications.ts)
```typescript
// שליחה לכל 4 ערוצים בקריאה אחת:
export const notify = async (userId: string, event: NotificationEvent) => {
  const user = await prisma.user.findUnique({ where: { id: userId } });
  const message = templates[event](user);

  await Promise.allSettled([
    sendWhatsApp(user.phone, message.whatsapp),
    sendSMS(user.phone, message.sms),
    sendEmail(user.email, message.email),
    sendPush(userId, message.push),
  ]);
};

// אירועים:
// LOAN_REQUESTED        → מנהל גמח
// LOAN_APPROVED         → משתמש
// LOAN_REJECTED         → משתמש
// LOAN_COLLECTED        → משתמש (אישור איסוף)
// LOAN_RETURN_REMINDER  → משתמש (יום לפני)
// LOAN_RETURNED_OK      → משתמש
// LOAN_OVERDUE          → משתמש + מנהל
// USER_REGISTERED       → מנהל ראשי
// USER_APPROVED         → משתמש
```

## 8. דפי האפליקציה

### 8.1 ברוכים הבאים (`/`)
- לוגו "המתבן" + רקע שדה חקלאי
- כפתור **"כניסה"** | כפתור **"הרשמה"** | כפתור **"כניסה כאורח"**
- אורח → קטלוג בצפייה בלבד (כפתור "בקש השאלה" → "יש להתחבר")

### 8.2 הרשמה (`/register`)
שדות חובה: **שם מלא | טלפון | אימייל | סיסמה**  
לאחר הרשמה → status=PENDING → הודעה: "בקשתך התקבלה, ממתין לאישור מנהל"  
מנהל ראשי מקבל WhatsApp על הרשמה חדשה.

### 8.3 קטלוג כלים (`/catalog`)
- כרטיסי כלים: תמונה, שם, גמח, זמינות (badge), תעריף יומי, פיקדון
- **חיפוש** טקסטואלי
- **פילטרים**: קטגוריה | גמח | זמינות | טווח תעריף
- לחיצה על כלי → דף כלי בודד

### 8.4 דף כלי (`/catalog/[toolId]`)
- גלריית תמונות
- פרטי כלי: תיאור, קטגוריה, גמח, תעריף יומי, סכום פיקדון, מקסימום ימים
- **יומן זמינות** חודשי
- כפתור **"בקש השאלה"** → LoanRequestForm (תאריכים + מטרה)
- **דירוגים וביקורות** (ציבוריות, שם מוצג)

### 8.5 ההשאלות שלי (`/my-loans`)
- טאבים: פעילות | ממתינות | היסטוריה
- לכל השאלה: שם כלי, תאריכים, סטטוס (badge), כפתור ביטול (אם PENDING)
- ציר זמן ויזואלי לכל השאלה פעילה

### 8.6 ניהול גמח (`/gemach/[id]/dashboard`) — מנהל גמח
**לשוניות:**
- **סקירה**: כלים פעילים, בקשות ממתינות, סטטיסטיקות
- **כלים**: רשימה + הוספה/עריכה/מחיקה + הגדרת autoApprove לכל כלי
- **בקשות**: טבלה + אישור/דחייה + הוספת הערה
- **החזרות**: אישור החזרה תקינה / דיווח נזק → שחרור/גביית פיקדון
- **הגדרות גמח**: שם, כתובת, טלפון, תמונה

### 8.7 לוח בקרה ראשי (`/admin`) — מנהל ראשי
**לשוניות:**
- **משתמשים**: אישור/דחייה/השעיה/ביטול חסימה
- **גמחים**: יצירה/עריכה + כניסה לניהול כל גמח
- **כלים**: הוספת כלים לכל גמח
- **סטטיסטיקות**: גרפים – השאלות, הכנסות, גמחים פעילים
